# 소개

서버리스 아키텍처는 클라우드 공급자가 서버를 실행하고 고객이 사용량에 따라 요금을 지불하는 클라우드 컴퓨팅 실행 모델입니다. 인프라를 관리할 필요 없이 애플리케이션과 서비스를 구축하고 실행하는 방법입니다.

서버리스 모델에서 개발자는 서버를 프로비저닝하거나 관리할 필요 없이 클라우드에서 실행되는 코드를 작성할 수 있습니다. 코드를 업로드하기만 하면 클라우드 공급자가 필요에 따라 코드를 실행합니다.

AWS Lambda는 이벤트에 대한 응답으로 코드를 실행하고 기본 컴퓨팅 리소스를 자동으로 관리하는 서버리스 컴퓨팅 서비스입니다. AWS Lambda를 사용하여 새로운 정보에 빠르게 응답하는 애플리케이션을 구축할 수 있습니다. Lambda는 Amazon DynamoDB 스트림의 레코드를 처리하고 AWS Lambda 함수를 호출할 수도 있습니다.

# 서버리스 아키텍처의 장점

서버리스 아키텍처의 주요 이점은 다음과 같습니다.

- 서버 관리 없음: 클라우드 공급자가 모든 인프라를 관리하므로 개발자가 훨씬 쉽게 코드에 집중할 수 있습니다.

- 비용 절감: 고객은 사용한 컴퓨팅 리소스에 대해서만 비용을 지불하므로 서버리스는 애플리케이션과 서비스를 실행하는 매우 비용 효율적인 방법입니다.

- 향상된 유연성: 서버를 프로비저닝하거나 관리할 필요가 없기 때문에 개발자는 필요에 따라 애플리케이션을 쉽게 확장하거나 축소할 수 있습니다.

- 효율성 향상: 개발자는 코드 작성에 집중하고 클라우드 공급자가 코드를 실행하는 데 필요한 모든 획일적인 작업을 처리하도록 할 수 있습니다.

# AWS 람다

AWS Lambda는 서버를 프로비저닝하거나 관리하지 않고도 코드를 실행할 수 있는 서버리스 컴퓨팅 서비스입니다. Lambda는 코드를 실행하는 데 필요한 모든 획일적인 작업을 처리합니다.

Lambda는 각 트리거에 대한 응답으로 코드를 실행하여 애플리케이션을 자동으로 확장합니다. Lambda는 고가용성 컴퓨팅 인프라에서 코드를 실행하고 서버 및 운영 체제 유지 관리, 용량 프로비저닝 및 자동 확장을 포함하여 컴퓨팅 리소스의 모든 관리를 수행합니다.

Lambda는 이벤트 기반 애플리케이션에 적합하며 다음과 같은 다양한 작업에 사용할 수 있습니다.

- 이미지 처리: 모바일 장치용으로 즉시 이미지 크기를 조정합니다.

- 파일 변환: 비디오 파일을 한 형식에서 다른 형식으로 변환합니다.

- 로그 처리: Amazon S3에서 로그 파일을 처리하고 분석합니다.

- 데이터 처리: Amazon Kinesis에서 스트리밍 데이터를 처리합니다.

- IoT 백엔드: Amazon IoT 장치에서 IoT 데이터를 처리하고 라우팅합니다.
