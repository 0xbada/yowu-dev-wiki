  # AWS Lambda: 이벤트 기반 애플리케이션을 위한 서버리스 기능 구축 AWS Lambda는 이벤트에 대한 응답으로 코드를 실행하고 기본 컴퓨팅 리소스를 자동으로 관리하는 서버리스 컴퓨팅 서비스입니다. AWS Lambda를 사용하여 새로운 정보에 빠르게 응답하는 애플리케이션을 구축할 수 있습니다. 예를 들어 이벤트 기반 애플리케이션은 Amazon Simple Storage Service(Amazon S3) 또는 Amazon Kinesis에서 제공하는 로그 파일을 처리하고 Amazon DynamoDB에 로그 데이터 저장, Amazon Simple Notification Service(Amazon SNS)를 통해 알림 전송과 같은 작업을 수행할 수 있습니다. ) 또는 AWS Lambda 함수를 호출하여 추가 처리를 수행합니다. Lambda를 사용하면 관리 없이 거의 모든 유형의 애플리케이션 또는 백엔드 서비스에 대한 코드를 실행할 수 있습니다. 코드를 업로드하기만 하면 Lambda가 고가용성으로 코드를 실행하고 확장하는 데 필요한 모든 것을 처리합니다. 다른 AWS 서비스에서 자동으로 트리거하거나 웹 또는 모바일 앱에서 직접 호출하도록 코드를 설정할 수 있습니다. Lambda는 고가용성 컴퓨팅 인프라에서 코드를 실행하고 서버 및 운영 체제 유지 관리, 용량 프로비저닝, 코드 모니터링 및 로깅을 포함하여 컴퓨팅 리소스의 모든 관리를 수행합니다. 코드를 제공하기만 하면 됩니다. ## AWS Lambda 시작하기 AWS Lambda를 시작하려면 Lambda 함수를 생성한 다음 함수에 대한 이벤트 소스를 구성합니다. 이벤트 소스는 AWS Lambda 함수 실행을 트리거하는 이벤트를 생성하는 AWS 서비스 또는 개발자가 만든 애플리케이션입니다. 예를 들어 Amazon S3 버킷에서 객체가 생성될 때, 메시지가 Amazon Simple Notification Service(Amazon SNS) 주제에 게시될 때 또는 Amazon DynamoDB 테이블이 업데이트될 때 Lambda 함수가 트리거될 수 있습니다. Lambda 함수를 생성할 때 함수를 트리거하는 이벤트와 이벤트를 처리하는 코드를 지정합니다. 지정된 이벤트 중 하나가 발생하면 Lambda가 코드를 실행합니다. Lambda는 사전 정의된 이벤트 소스 세트를 제공하며 고유한 이벤트 소스를 추가할 수도 있습니다. Lambda 함수를 생성하려면: 1. Lambda 콘솔을 엽니다. 2. 탐색 패널에서 기능을 선택합니다. 3. 함수 생성을 선택합니다. 4. 처음부터 작성을 선택합니다. 5. 함수 이름 필드에 함수 이름을 입력합니다. 6. 런타임 드롭다운에서 함수의 언어를 선택합니다. 7. 역할에서 기존 역할 선택을 선택합니다. 8. 기존 역할 드롭다운에서 Amazon S3 또는 Amazon DynamoDB와 같은 계정의 리소스에 액세스할 수 있는 Lambda 권한을 부여하는 역할을 선택합니다. 자세한 내용은 Lambda 함수에 대한 역할 생성을 참조하십시오. 9. 함수 생성을 선택합니다. 함수가 생성되어 함수 목록에 표시됩니다. ## 이벤트 소스 구성 Lambda 함수를 생성한 후 함수에 대한 이벤트 소스를 구성해야 합니다. 이벤트 소스를 구성할 때 이벤트 소스 유형과 해당 이벤트 소스 유형에 필요한 매개변수를 지정합니다. 예를 들어 Amazon Kinesis 스트림을 이벤트 소스로 사용하는 경우 Amazon Kinesis 스트림 이름과 Arn을 지정해야 합니다. Lambda는 이벤트 소스의 이벤트를 순차적으로 처리합니다. 이벤트를 병렬로 처리하려면 여러 개의 동일한 Lambda 함수를 생성하고 각 함수를 서로 다른 이벤트 소스로 구성합니다. 자세한 내용은 Amazon Kinesis와 함께 AWS Lambda 사용을 참조하십시오. 함수를 만들 때 이벤트 소스를 구성하거나 나중에 이벤트 소스를 추가할 수 있습니다. 함수를 생성할 때 이벤트 소스를 구성하려면: 1. Lambda 콘솔을 엽니다. 2. 탐색 패널에서 기능을 선택합니다. 3. 이벤트 소스를 추가하려는 함수의 이름을 선택합니다. 4. 함수 코드 편집기에서 코드를 입력합니다. 5. 디자이너에서 트리거 추가를 선택합니다. 6. 트리거 구성 섹션에서 이벤트 소스 유형을 선택합니다. 7. 이벤트 소스에서 이벤트 소스를 선택합니다. 8. 이벤트 소스를 구성합니다. 9. 추가를 선택합니다. ## Lambda 함수 코드 Lambda 함수는 선택한 언어로 작성됩니다. Lambda는 AWS SDK를 포함하는 각 언어에 대해 미리 정의된 환경 세트를 제공합니다. AWS SDK를 사용하면 Lambda 함수 코드에서 AWS 서비스를 호출할 수 있습니다. 예를 들어 Amazon S3를 사용하여 데이터를 저장하고 Amazon DynamoDB를 사용하여 데이터를 읽고 쓰거나 Amazon Simple Queue Service(Amazon SQS)를 사용하여 메시지를 보내고 받을 수 있습니다. Lambda는 또한 AWS SDK에서 사용할 수 없는 서비스를 호출하거나 Lambda 콘솔에서 사용할 수 없는 AWS SDK 작업을 호출하는 데 사용할 수 있는 런타임 API를 제공합니다. 자세한 내용은 AWS Lambda 런타임 API를 참조하십시오. ## Lambda 함수 핸들러 핸들러는 Lambda 함수의 진입점입니다. IAM 역할을 생성하고 함수가 액세스할 수 있는 리소스를 식별하는 정책을 연결하여 이를 수행합니다. 자세한 내용은 Lambda 함수에 대한 역할 생성을 참조하십시오. ## Lambda 함수 정책 Lambda 함수를 생성할 때 지정하는 IAM 역할 외에도 Lambda는 함수의 IAM 역할에 정책을 연결합니다. 이 정책은 Amazon CloudWatch Logs에 이벤트를 기록하고 추적 데이터를 X-Ray에 쓸 수 있는 함수 권한을 부여합니다. 자세한 내용은 CloudWatch Logs에 Lambda 함수 활동 로깅 단원을 참조하십시오. ## Lambda 함수 실행 역할 Lambda 함수를 생성할 때 실행 역할을 지정합니다. 실행 역할은 함수가 작업을 수행하는 데 필요한 AWS 리소스에 액세스할 수 있는 Lambda 권한을 부여합니다. 예를 들어 Lambda 함수가 Amazon S3 버킷 또는 Amazon DynamoDB 테이블에서 데이터를 읽고 쓰는 경우 해당 리소스에 액세스할 수 있는 함수 권한을 부여해야 합니다. IAM 역할을 생성하고 함수가 액세스할 수 있는 리소스를 식별하는 정책을 연결하여 이를 수행합니다. 자세한 내용은 Lambda 함수에 대한 역할 생성을 참조하십시오. ## Lambda 함수 정책 Lambda 함수를 생성할 때 지정하는 IAM 역할 외에도 Lambda는 함수의 IAM 역할에 정책을 연결합니다. 이 정책은 Amazon CloudWatch Logs에 이벤트를 기록하고 추적 데이터를 X-Ray에 쓸 수 있는 함수 권한을 부여합니다. 자세한 내용은 CloudWatch Logs에 Lambda 함수 활동 로깅 단원을 참조하십시오. ## Lambda 함수 실행 역할 Lambda 함수를 생성할 때 실행 역할을 지정합니다. 실행 역할은 함수가 작업을 수행하는 데 필요한 AWS 리소스에 액세스할 수 있는 Lambda 권한을 부여합니다. 예를 들어 Lambda 함수가 Amazon S3 버킷 또는 Amazon DynamoDB 테이블에서 데이터를 읽고 쓰는 경우 해당 리소스에 액세스할 수 있는 함수 권한을 부여해야 합니다.
