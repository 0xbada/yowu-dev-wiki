  # DNS(도메인 이름 시스템) 이해 DNS(도메인 이름 시스템)는 인터넷에 연결된 장치와 장치가 제공하는 리소스의 디렉토리입니다. 인간이 읽을 수 있는 도메인 이름(예: "google.com")을 컴퓨터가 서로 통신하는 데 사용하는 IP 주소(예: "216.58.221.46")로 변환하여 인간이 인터넷과 상호 작용하는 방식입니다. DNS는 인터넷의 중요한 구성 요소입니다. 그것은 우리가 웹사이트의 주소를 입력하고 올바른 서버로 안내할 수 있게 해줍니다. 또한 이메일 전송을 가능하게 하고 프린터 및 스마트 TV와 같은 인터넷 지원 장치에 연결할 수 있습니다. 웹 브라우저에 도메인 이름을 입력하면 컴퓨터가 DNS 서버에 연결하여 해당 도메인과 연결된 IP 주소를 요청합니다. 그런 다음 DNS 서버는 IP 주소로 응답하고 컴퓨터는 해당 주소의 서버에 연결됩니다. DNS 서버는 루트 DNS 서버가 맨 위에 있는 계층 구조로 구성됩니다. 계층 구조의 각 수준에는 DNS 시스템의 IP 주소 하위 집합에 대해 알고 있는 DNS 서버가 포함되어 있습니다. 예를 들어 루트 DNS 서버는 다음 수준 아래에 있는 DNS 서버의 IP 주소만 알고 있습니다. 다음 수준 아래에 있는 DNS 서버는 그 다음 수준에 있는 DNS 서버의 IP 주소만 알고 있습니다. 이 계층 구조를 통해 DNS 서버는 요청에 응답하는 로드를 공유할 수 있습니다. 또한 계층 구조의 일부 서버를 사용할 수 없는 경우에도 DNS 서버가 계속 작동할 수 있습니다. 계층 구조의 각 수준에 있는 DNS 서버에는 다음 수준 아래에 있는 DNS 서버의 IP 주소를 지정하는 레코드가 포함되어 있습니다. 이러한 레코드를 DNS 레코드라고 하며 영역으로 구성됩니다. 영역은 특정 DNS 서버에서 관리하는 DNS 네임스페이스의 일부입니다. 각 DNS 서버는 특정 영역에 대한 레코드 유지 관리를 담당합니다. 예를 들어 루트 DNS 서버는 루트 영역에 대한 레코드 유지 관리를 담당합니다. 다음 수준 아래에 있는 DNS 서버는 최상위 도메인 등에 대한 레코드를 유지 관리하는 역할을 합니다. DNS 서버는 자체 영역에 없는 DNS 레코드에 대한 요청에 응답하기 위해 재귀 확인이라는 프로세스를 사용합니다. DNS 서버가 가지고 있지 않은 레코드에 대한 요청을 받으면 레코드가 있는 DNS 서버에 접속하여 레코드를 요청합니다. 레코드가 있는 DNS 서버는 레코드로 응답하고 요청을 한 DNS 서버는 레코드를 원래 요청자에게 다시 전달합니다. DNS 서버는 반복 확인이라는 프로세스를 사용하여 DNS 레코드 요청에 응답할 수도 있습니다. 반복 확인을 통해 DNS 서버는 레코드가 있는 DNS 서버에 연결하여 레코드를 요청합니다. 레코드가 있는 DNS 서버는 레코드로 응답합니다. DNS 서버는 재귀 및 반복 확인의 조합을 사용하여 DNS 레코드에 대한 요청에 응답합니다. DNS 서버는 DNS 레코드의 캐시를 유지 관리합니다. DNS 서버가 레코드에 대한 요청을 받으면 캐시를 확인하여 이미 레코드가 있는지 확인합니다. DNS 서버의 캐시에 레코드가 있으면 캐시의 레코드로 요청에 응답합니다. DNS 서버의 캐시에 레코드가 없으면 레코드가 있는 DNS 서버에 연결하여 레코드를 요청합니다. 레코드가 있는 DNS 서버는 레코드로 응답하고 요청을 한 DNS 서버는 레코드를 원래 요청자에게 다시 전달합니다. 또한 DNS 서버는 DNS 서버에 연결할 필요 없이 레코드에 대한 향후 요청에 응답할 수 있도록 레코드를 캐시에 추가합니다. DNS 서버는 캐싱이라는 프로세스를 사용하여 DNS 시스템의 성능을 향상시킵니다. DNS 레코드를 캐싱함으로써 DNS 서버는 다른 DNS 서버에 연결할 필요 없이 레코드 요청에 응답할 수 있습니다. 캐싱은 또한 DNS 서버와 네트워크의 부하를 줄입니다. DNS 서버가 레코드를 캐시하면 다른 DNS 서버에 연결할 필요 없이 레코드에 대한 향후 요청에 응답할 수 있습니다. DNS 서버는 재귀 또는 반복 확인자로 작동하도록 구성할 수 있습니다. 재귀 확인자는 클라이언트를 대신하여 다른 DNS 서버에 연결하고 DNS 레코드를 검색하도록 구성된 DNS 서버입니다. 반복 확인자는 다른 DNS 서버에 연결하고 DNS 레코드를 요청하도록 구성된 DNS 서버이지만 클라이언트를 대신하여 레코드를 검색하지는 않습니다. 재귀 또는 반복 확인을 사용하도록 DNS 서버를 구성할 수 있습니다. 대부분의 DNS 서버는 재귀 확인자로 구성됩니다. 재귀 확인자는 반복 확인자보다 구성 및 관리가 더 쉽습니다. 반복 해석기는 DNS 시스템과 네트워크에 대한 더 많은 지식이 필요합니다. 재귀 확인자는 반복 확인자보다 더 빠르게 DNS 쿼리를 해결할 수 있으므로 더 나은 사용자 경험을 제공할 수 있습니다. 반복 확인자는 DNS 쿼리를 해결하기 위해 여러 DNS 서버에 연결해야 하며 시간이 더 오래 걸릴 수 있습니다. 반복 확인자는 DNS 시스템과 네트워크에 대한 많은 정보를 공개하지 않기 때문에 재귀 확인자보다 더 안전할 수 있습니다. 재귀 및 반복 확인을 모두 사용하도록 DNS 서버를 구성할 수 있습니다. UDP 또는 TCP를 사용하도록 DNS 서버를 구성할 수 있습니다. UDP는 연결 없는 프로토콜이며 UDP를 사용하는 DNS 서버는 쿼리 중인 DNS 서버와 연결을 설정하지 않습니다. TCP는 연결 지향 프로토콜이며 TCP를 사용하는 DNS 서버는 쿼리 중인 DNS 서버와 연결을 설정합니다. UDP 또는 TCP를 사용하도록 DNS 서버를 구성할 수 있습니다. UDP는 TCP보다 빠르기 때문에 DNS에 선호되는 프로토콜입니다. UDP는 연결을 설정하지 않으므로 TCP의 오버헤드 없이 DNS 쿼리 및 응답을 보내고 받을 수 있습니다. TCP는 신뢰할 수 있는 프로토콜이며 TCP를 사용하는 DNS 서버는 응답을 받지 못한 경우 DNS 쿼리를 다시 시도할 수 있습니다. 방화벽을 통해 DNS 쿼리 및 응답을 터널링하는 데 TCP를 사용할 수도 있습니다. UDP와 TCP를 모두 사용하도록 DNS 서버를 구성할 수 있습니다. 다른 포트를 사용하도록 DNS 서버를 구성할 수 있습니다.
