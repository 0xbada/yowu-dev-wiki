# 了解域名系统 (DNS) 域名系统 (DNS) 是连接互联网的设备及其提供的资源的目录。这是人类通过将人类可读的域名（如“google.com”）翻译成计算机用来相互通信的 IP 地址（如“216.58.221.46”）来与互联网交互的方式。 DNS 是互联网的重要组成部分。它允许我们输入网站地址并被定向到正确的服务器。它也是实现电子邮件传递的原因，并且可以连接到支持互联网的设备，如打印机和智能电视。当您在 Web 浏览器中键入域名时，您的计算机会联系 DNS 服务器并向其询问与该域关联的 IP 地址。然后 DNS 服务器以 IP 地址响应，您的计算机连接到该地址的服务器。 DNS 服务器被组织成层次结构，根 DNS 服务器位于顶部。层次结构的每个级别都包含了解 DNS 系统中 IP 地址子集的 DNS 服务器。例如，根DNS服务器只知道下一级DNS服务器的IP地址。下一级DNS服务器只知道下一级DNS服务器的IP地址，以此类推。这种层次结构使 DNS 服务器可以分担响应请求的负载。它还使 DNS 服务器可以继续工作，即使层次结构中的某些服务器不可用。层次结构中每一层的 DNS 服务器都包含指定下一层 DNS 服务器 IP 地址的记录。这些记录称为 DNS 记录，它们被组织成区域。区域是由特定 DNS 服务器管理的 DNS 名称空间的一部分。每个 DNS 服务器负责维护特定区域的记录。例如，根 DNS 服务器负责维护根区域的记录。下一级的 DNS 服务器负责维护顶级域的记录等。 DNS 服务器使用称为递归解析的过程来响应对它们自己区域中没有的 DNS 记录的请求。当 DNS 服务器收到对它没有的记录的请求时，它会联系确实有该记录的 DNS 服务器并请求该记录。拥有该记录的 DNS 服务器随后以该记录进行响应，发出请求的 DNS 服务器将记录传回给原始请求者。 DNS 服务器还可以使用称为迭代解析的过程来响应对 DNS 记录的请求。通过迭代解析，DNS 服务器联系具有记录的 DNS 服务器并请求记录。拥有该记录的 DNS 服务器随后以该记录进行响应。 DNS 服务器结合使用递归和迭代解析来响应对 DNS 记录的请求。 DNS 服务器维护 DNS 记录的缓存。当 DNS 服务器收到对记录的请求时，它会检查其缓存以查看它是否已经拥有该记录。如果 DNS 服务器在其缓存中有记录，它会使用其缓存中的记录响应请求。如果 DNS 服务器在其缓存中没有记录，它会联系有记录的 DNS 服务器并请求记录。拥有该记录的 DNS 服务器随后以该记录进行响应，发出请求的 DNS 服务器将记录传回给原始请求者。 DNS 服务器还将记录添加到其缓存中，以便它可以响应未来的记录请求而无需联系 DNS 服务器。 DNS 服务器使用称为缓存的过程来提高 DNS 系统的性能。通过缓存 DNS 记录，DNS 服务器可以响应记录请求而无需联系其他 DNS 服务器。缓存还可以减少 DNS 服务器和网络的负载。当 DNS 服务器缓存一条记录时，它可以响应将来对该记录的请求，而无需联系其他 DNS 服务器。 DNS 服务器可以配置为充当递归或迭代解析器。递归解析器是配置为联系其他 DNS 服务器并代表其客户端检索 DNS 记录的 DNS 服务器。迭代解析器是配置为联系其他 DNS 服务器并请求 DNS 记录的 DNS 服务器，但它们不代表其客户端检索记录。 DNS 服务器可以配置为使用递归或迭代解析。大多数 DNS 服务器都配置为递归解析器。递归解析器比迭代解析器更容易配置和管理。迭代解析器需要更多的 DNS 系统和网络知识。递归解析器可以提供更好的用户体验，因为它们可以比迭代解析器更快地解析 DNS 查询。迭代解析器必须联系多个 DNS 服务器来解析 DNS 查询，这可能需要更长的时间。迭代解析器可能比递归解析器更安全，因为它们不会泄露尽可能多的有关 DNS 系统和网络的信息。 DNS 服务器可以配置为同时使用递归和迭代解析。 DNS 服务器可以配置为使用 UDP 或 TCP。 UDP 是一种无连接协议，使用 UDP 的 DNS 服务器不会与它们正在查询的 DNS 服务器建立连接。 TCP 是一种面向连接的协议，使用 TCP 的 DNS 服务器与它们正在查询的 DNS 服务器建立连接。 DNS 服务器可以配置为使用 UDP 或 TCP。 UDP 是 DNS 的首选协议，因为它比 TCP 更快。 UDP 不建立连接，因此它可以发送和接收 DNS 查询和响应，而无需 TCP 的开销。 TCP 是一种可靠的协议，使用 TCP 的 DNS 服务器可以在未收到响应时重试 DNS 查询。 TCP 还可用于通过防火墙隧道 DNS 查询和响应。 DNS 服务器可以配置为同时使用 UDP 和 TCP。 DNS 服务器可以配置为使用不同的端口。配置为使用 UDP 的 DNS 服务器通常配置为使用端口 53。配置为使用 TCP 的 DNS 服务器通常配置为使用端口 53。DNS 服务器可以配置为使用不同的端口。 DNS 服务器可以配置为对 UDP 和 TCP 使用不同的端口。配置为使用 UDP 的 DNS 服务器通常配置为使用端口 53。配置为使用 TCP 的 DNS 服务器通常配置为使用端口 53。DNS 服务器可以配置为对 UDP 和 TCP 使用不同的端口。 DNS 服务器可以配置为转发 DNS 查询。配置为转发 DNS 查询的 DNS 服务器称为转发器。转发器是配置为向其他 DNS 服务器发送 DNS 查询的 DNS 服务器。转发器可用于提高 DNS 系统的性能。配置为使用转发器的 DNS 服务器可以将 DNS 查询发送到多个 DNS 服务器。这可以提高 DNS 系统的性能，因为配置为使用转发器的 DNS 服务器可以比未配置为使用转发器的 DNS 服务器更快地解析 DNS 查询。配置为使用转发器的 DNS 服务器也可用于提高 DNS 系统的安全性。配置为使用转发器的 DNS 服务器可以将 DNS 查询发送到位于不同网络中的 DNS 服务器。这可以提高 DNS 系统的安全性，因为配置为使用转发器的 DNS 服务器可以解析 DNS 查询，而无需联系位于同一网络中的 DNS 服务器。 DNS 服务器可以配置为使用转发器。 DNS
