  # Comprender el Sistema de nombres de dominio (DNS) El Sistema de nombres de dominio (DNS) es un directorio de dispositivos conectados a Internet y los recursos que ofrecen. Es la forma en que los humanos interactúan con Internet mediante la traducción de nombres de dominio legibles por humanos (como "google.com") en las direcciones IP que usan las computadoras para comunicarse entre sí (como "216.58.221.46"). El DNS es un componente crítico de Internet. Es lo que nos permite escribir la dirección de un sitio web y ser dirigidos al servidor correcto. También es lo que permite la entrega de correo electrónico y hace posible conectarse a dispositivos habilitados para Internet, como impresoras y televisores inteligentes. Cuando escribe un nombre de dominio en su navegador web, su computadora se comunica con un servidor DNS y le solicita la dirección IP asociada con ese dominio. Luego, el servidor DNS responde con la dirección IP y su computadora se conecta al servidor en esa dirección. Los servidores DNS están organizados en una jerarquía, con los servidores DNS raíz en la parte superior. Cada nivel de la jerarquía contiene servidores DNS que conocen un subconjunto de direcciones IP en el sistema DNS. Por ejemplo, los servidores DNS raíz solo conocen las direcciones IP de los servidores DNS del siguiente nivel. Los servidores DNS del siguiente nivel inferior solo conocen las direcciones IP de los servidores DNS del nivel posterior, y así sucesivamente. Esta jerarquía hace posible que los servidores DNS compartan la carga de responder a las solicitudes. También hace posible que los servidores DNS continúen funcionando incluso si algunos de los servidores de la jerarquía no están disponibles. Los servidores DNS en cada nivel de la jerarquía contienen registros que especifican las direcciones IP de los servidores DNS en el siguiente nivel. Estos registros se denominan registros DNS y están organizados en zonas. Una zona es una parte del espacio de nombres DNS que administra un servidor DNS en particular. Cada servidor DNS es responsable de mantener los registros de una zona en particular. Por ejemplo, los servidores DNS raíz son responsables de mantener los registros de la zona raíz. Los servidores DNS del siguiente nivel inferior son responsables de mantener los registros de los dominios de nivel superior, y así sucesivamente. Los servidores DNS utilizan un proceso llamado resolución recursiva para responder a las solicitudes de registros DNS que no tienen en sus propias zonas. Cuando un servidor DNS recibe una solicitud de un registro que no tiene, se comunica con un servidor DNS que sí tiene el registro y solicita el registro. El servidor DNS que tiene el registro responde con el registro y el servidor DNS que realizó la solicitud devuelve el registro al solicitante original. Los servidores DNS también pueden usar un proceso llamado resolución iterativa para responder a las solicitudes de registros DNS. Con resolución iterativa, un servidor DNS contacta a un servidor DNS que tiene el registro y solicita el registro. El servidor DNS que tiene el registro responde con el registro. Los servidores DNS utilizan una combinación de resolución recursiva e iterativa para responder a las solicitudes de registros DNS. Los servidores DNS mantienen un caché de registros DNS. Cuando un servidor DNS recibe una solicitud de registro, verifica su caché para ver si ya tiene el registro. Los servidores DNS utilizan un proceso llamado almacenamiento en caché para mejorar el rendimiento del sistema DNS. Al almacenar en caché los registros DNS, los servidores DNS pueden responder a las solicitudes de registros sin tener que ponerse en contacto con otros servidores DNS. El almacenamiento en caché también reduce la carga en los servidores DNS y la red.
